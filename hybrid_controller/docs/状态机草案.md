# 状态机草案

统一总控当前使用的主状态如下：

- `IDLE`
- `S1_MI_MOVE`
- `S1_DECISION`
- `S2_TARGET_SELECT`
- `S2_GRAB_CONFIRM`
- `S2_PICKING`
- `S3_MI_CARRY`
- `S3_DECISION`
- `S3_PLACING`
- `FINISHED`
- `ERROR`

## 主流程

1. `IDLE -> S1_MI_MOVE`
2. `S1_MI_MOVE -> S1_DECISION`
3. `S1_DECISION -> S2_TARGET_SELECT`
4. `S2_TARGET_SELECT -> S2_GRAB_CONFIRM`
5. `S2_GRAB_CONFIRM -> S2_PICKING`
6. `S2_PICKING -> S3_MI_CARRY`
7. `S3_MI_CARRY -> S3_DECISION`
8. `S3_DECISION -> S3_PLACING`
9. `S3_PLACING -> FINISHED`

## 关键原则

- 状态机不直接碰硬件。
- 所有输入先转统一事件。
- UI 不直接发 socket。
- 抓取和放置推进只靠 `ACK PICK_DONE` / `ACK PLACE_DONE`。
- 机器人错误在执行态进入 `ERROR`，通过 `RESET` 恢复。

# 主线入口与代码地图

本文件用于快速说明 `brain_code` 中每个部分应该从哪里进入、当前哪个目录是主线、哪个目录只是历史参考。

## 1. 总体结构

`brain_code` 当前分为五个代码类别和一个文档目录：

- `01_MI`
- `02_SSVEP`
- `03_RobotArm_Control`
- `04_Communication_And_Integration`
- `05_Vision_Block_Recognition`
- `docs`

## 2. 当前主线文件

如果只看现在最值得继续维护的文件，建议优先看下面这些：

### 2.1 总控原型

- `05_Vision_Block_Recognition/2026-03_yolo_camera_detection/computer/test2.py`

作用：

- 当前最接近完整任务流程的总控原型
- 已有搜索、抓取、搬运、放置状态骨架
- 已能向机械臂发送 `MOVE`、`PICK`、`PLACE`

### 2.2 机械臂执行端

- `03_RobotArm_Control/2026-03_jetmax_execution_server/test2_robot.py`

作用：

- 当前机械臂执行端主脚本
- 负责解析 socket 命令并执行移动、抓取、放置
- 应被视为统一通信协议参考实现

### 2.3 SSVEP 主线

- `02_SSVEP/2026-03_realtime_ui_and_online_decoder/SSVEP/demo.py`

作用：

- 当前最新的实时刺激和在线识别实现
- 含 FBCCA、实时窗口分析、平滑输出和命令信号

### 2.4 MI 主线

- `01_MI/mi_classifier_latest/code/realtime/mi_realtime_infer_only.py`
- `01_MI/mi_classifier_latest/code/shared/src/realtime_mi.py`

作用：

- 当前 MI 实时推理主线
- 提供稳定输出、gate、artifact reject 和在线决策状态

## 3. 各分类目录说明

### 3.1 `01_MI`

主线目录：

- `mi_classifier_latest/`

常用子目录：

- `code/collection/`
- `code/training/`
- `code/realtime/`
- `code/viewer/`

说明：

- 这是当前最完整的 MI 仓库
- 已具备采集、训练、实时推理完整链路
- `datasets/` 和 `runtime/` 属于本地数据与运行产物，不作为仓库主线内容上传

### 3.2 `02_SSVEP`

主线目录：

- `2026-03_realtime_ui_and_online_decoder/`

历史参考目录：

- `2026-02_realtime_stimulus_and_classifier_core/`
- `2026-02_algorithms_and_data_tools/`
- `2026-02_stimulus_variants/`
- `2026-02_custom_dataset_scripts/`

说明：

- `2026-03` 更适合继续做在线识别集成
- `2026-02` 更适合作为算法细节和旧实现参考

### 3.3 `03_RobotArm_Control`

主线目录：

- `2026-03_jetmax_execution_server/`

说明：

- 这是机械臂执行端主线
- 后续所有联调代码都应围绕这里的协议来统一

### 3.4 `04_Communication_And_Integration`

当前用途：

- 保存 socket 客户端、抓取命令脚本和调试脚本

说明：

- 这里更像历史联调区和工具区
- 不建议再把它继续扩展成新的系统主入口

### 3.5 `05_Vision_Block_Recognition`

主线目录：

- `2026-03_yolo_camera_detection/`

历史参考目录：

- `2026-02_template_matching_and_camera/`

说明：

- 这是当前最适合继续扩展为总控主程序的目录
- 其中 `computer/test2.py` 是当前最有价值的系统骨架

## 5. 当前建议的主线开发路线

从仓库结构角度看，后续最合理的主线是：

1. 以 `05_Vision_Block_Recognition/2026-03_yolo_camera_detection/computer/test2.py` 为总控入口
2. 以 `03_RobotArm_Control/2026-03_jetmax_execution_server/test2_robot.py` 为机械臂执行端
3. 以 `02_SSVEP/2026-03_realtime_ui_and_online_decoder/SSVEP/demo.py` 为 SSVEP 解码入口
4. 以 `01_MI/mi_classifier_latest/code/realtime/mi_realtime_infer_only.py` 为 MI 解码入口

这四个位置应该被看作当前真正的系统主线。
